# 开发日志 - 2026-07-03

## 今日完成

### 多 SMB 服务器支持

**新增文件:**
- `SmbServerConfig.java` — 服务器配置 POJO, fastjson 序列化 (密码不参与 JSON)
- `SmbServerListActivity.java` — 服务器管理列表页
- `activity_smb_server_list.xml` — 服务器列表布局

**Settings.java — 多服务器存储:**
- 新增 `smb_servers` (JSON 数组, fastjson 序列化, 不含密码)
- 新增 `smb_active_index` (int, 默认为 0)
- 新增 `smb_auto_switch` (boolean, 默认为 false)
- 密码存储改为索引键: `smb_password_0`, `smb_password_1`...
- 新增 `getActiveSmbServer()` 便捷方法
- `getDownloadLocation()` 改为从活跃服务器配置创建 SmbFile
- 自动迁移旧单服务器配置到新 JSON 列表格式

**SmbConfigActivity.java — 双模式适配:**
- 添加模式 (默认): 空字段, 保存时 append 到服务器列表
- 编辑模式 (`EXTRA_SERVER_INDEX`): 预填字段, 保存时替换对应索引
- `selectCurrentDir()` 改为保存到 JSON 列表而非单键

**DownloadFragment.java:**
- "Configure SMB server" 按钮改为启动 SmbServerListActivity

### 2 秒超时健康检测

**SmbConnectionManager.java — 新增 `healthCheck()`:**
- 静态方法, 独立临时连接, 不干扰全局共享连接
- 超时: connect=1500ms, soTimeout=500ms (总计 ≤ 2s)
- 连接成功即视为可达 (移除原 `folderExists("/")` 探测)
- finally 块中确保所有 SmbJ 资源关闭

**MainActivity.java — `checkDownloadLocation()` 改造:**
- AsyncTask 中统一执行健康检测 + 故障切换 (全部在后台线程)
- 返回码: 1=当前正常, 2=已切换到其他服务器, 3=全部不可达
- `onPostExecute` 仅负责 UI 更新 (Toast/对话框)
- 新增 `ProgressDialog` 阻塞用户操作, 检测完成后自动关闭

### 故障切换逻辑

**故障切换循环修复:**
- 原代码 `for (int i = currentIndex + 1; ...)` 只尝试高索引服务器 → 导致列表最后一个服务器不可达时无法切换
- 修复为 `for (int i = 0; ...) { if (i == currentIndex) continue; }` 遍历所有其他服务器

**自动切换:**
- 设置项 `smb_auto_switch` 控制是否自动切换下一台服务器
- 自动切换关: 弹出对话框让用户选择 (切换到下一台 / 关闭 SMB / 取消)
- 自动切换开: 静默切换, 全部不可达时自动关闭 SMB

### 并发安全修复

**SmbFile.getShare() — 延迟清理旧连接:**
- 凭据不匹配时不再立即调用 `sSharedManager.disconnect()` 关闭旧连接
- 改为先交换引用指向新连接, 通过 `Handler.postDelayed` 15 秒后清理旧连接
- 给正在进行的读取/下载操作留出完成时间

**MainActivity `onPostExecute` 移除 `disconnectShared()`:**
- 异步检测结果不再主动断开共享连接
- 避免健康检查完成后打断正在进行的 SMB 读取操作

## 已知限制

- 随机访问: `createRandomAccessFile()` 抛出 `UnsupportedOperationException` (SMB 协议不支持)
- 中文路径: SMB 路径包含中文时需确保 SMB 服务器使用 UTF-8
- 性能: 每个 SMB 操作串行执行, 多线程下载时是瓶颈
- 图片读取: 每张图片从 SMB 全量下载到临时文件后再解码 (非流式)
- 旧连接延迟 15 秒清理期间可能短暂占用资源
