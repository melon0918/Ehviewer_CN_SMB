**Step 1.3 — SmbFile (UniFile 实现)**
- 包: `com.hippo.unifile.SmbFile`
- 实现所有 22 个 UniFile 抽象方法
- 关键 SmbJ API 适配 (0.13.0):
  - `DiskShare` 没有直接的 `openInputStream(String)` / `openOutputStream(String)` → 必须用 `openFile(path, ...)` 返回 `File` 对象，再调 `getInputStream()` / `getOutputStream()`
  - `DiskShare` 没有 `rename(String, String)` → 用 `openFile(path)` + `DiskEntry.rename(newPath)`
  - `DiskShare` 没有 `getFileSize(String)` → 用 `getFileInformation(path, FileStandardInformation.class).getEndOfFile()`
  - `DiskShare.list(String)` 返回 `List<FileIdBothDirectoryInformation>` 而非 `List<File>`
  - 所有文件信息查询方法抛 `SMBApiException` (继承 `SMBRuntimeException` → `RuntimeException`)，不是 `IOException`
- 自动清理: 流关闭时自动关闭底层 File handle

**Step 1.4 — SmbUriHandler**
- 包: `com.hippo.unifile.SmbUriHandler`, 实现 `UriHandler` 接口
- 解析 `smb://host/share/path` 和 `smb://user:pass@host/share/path`
- 注册: `EhApplication.onCreate()` 中 `UniFile.addUriHandler(new SmbUriHandler())`

### Phase 3: 集成与 Bug 修复

**Step 3.1 — 下载流程验证**
- 下载工作通过 UniFile 抽象层自动路由，无需修改 SpiderDen/DownloadManager
- `Settings.getDownloadLocation()` → `SmbFile` → `subFile` → `ensureDir` → `createFile` → `openOutputStream`

**Bug 1: ensureDir() 路径加倍**
- 症状: `STATUS_OBJECT_PATH_NOT_FOUND` — 路径如 `.../gallery/gallery`
- 原因: `ensureDir()` 调用 `createDirectory(getName())`, 而 `createDirectory()` 用 `getChildPath(name)` = `mPath + "/" + name`, 即在当前路径下创建同名子目录
- 修复: 改为直接 `share.mkdir(mPath)`, 保留递归父目录创建逻辑

**Bug 2: 连接风暴 (STATUS_REQUEST_NOT_ACCEPTED)**
- 症状: 多线程下载时 `Authentication failed`, SMB 服务器拒绝并发认证
- 原因: 每个 SmbFile 方法创建独立 SMB 连接，多个 SpiderWorker 同时认证
- 修复: 共享全局 `SmbConnectionManager`, 所有 SmbFile 实例复用同一个 DiskShare
- 线程安全: 所有共享 DiskShare 的操作用 `synchronized(sLock)` 串行化
- `getShare()`: 首次创建连接，后续复用 (按 host/share/username/password 匹配)
- 移除了 per-operation 的 closeShare() 调用

**Bug 3: Image.decode() 需要 FileInputStream**
- 症状: `ClassCastException: SmbFileInputStream cannot be cast to FileInputStream`
- 原因: SpiderQueen 和 ImageBitmapHelper 中 `(FileInputStream) pipe.open()`, Image.decode() 是 JNI 调用，需要文件描述符
- 修复: `openInputStream()` 改为从 SMB 下载到临时文件，返回 `FileInputStream`
- 临时文件位置: `context.getCacheDir()/smb_XXXXX.img` (通过 `SmbFile.setTempDir()` 设置)
- 临时文件清理: 流 close() 时自动删除, 启动时 `cleanTempFiles()` 清理残留

**Bug 4: NetworkOnMainThreadException**
- 症状: GalleryActivity 启动时崩溃，主线程调用 SmbFile 操作触发 StrictMode
- 原因: 原代码假设文件 I/O 在主线程是安全的，但 SMB 操作是网络 I/O
- 修复: `EhApplication.onCreate()` 中设置 `StrictMode.setThreadPolicy(detectAll().penaltyLog())`, 违规记日志不崩溃
- 辅助: SmbFile 每个方法也独立做 `relaxStrictMode()` (permitNetwork) 保护

**Bug 5: SmbConfigActivity 目录不显示**
- 原因1: 路径 `"/"` 传给 SmbJ 的 `share.list("/")` 产生双反斜杠 `\\*`
- 原因2: emoji "📁" 在某些设备不渲染
- 修复: 改用 `openDirectory("") + Directory.list()`, 路径用空字符串表示根目录
- 最终使用 ScrollView + LinearLayout + 手动添加 TextView (彻底绕过 ListView/RecyclerView 渲染问题)

## 已知限制

- 随机访问: `createRandomAccessFile()` 抛出 `UnsupportedOperationException` (SMB 协议不支持)
- 中文路径: SMB 路径包含中文时，确保 SMB 服务器使用 UTF-8
- 性能: 每个 SMB 操作串行执行，多线程下载时是瓶颈
- 图片读取: 每张图片从 SMB 全量下载到临时文件后再解码 (非流式)
